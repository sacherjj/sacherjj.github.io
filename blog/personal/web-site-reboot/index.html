Web Site Reboot

2014-05-05T18:00:00-04:00 Mon 5th May, 2014

1 min read

P1M 227

Category: personal

Tags:

  • site related

Tweet

It is hard for me to believe that I tried to go to a markdown or reStructured text driven web site over a year ago. I started with a Ruby based solutions, thinking it would be a good time to learn. But the only thing I really learned is that Ruby is a 2nd class citizen on Windows. I need to do 60% of my work in Windows and a VM can’t cut it.

How many times do you think I logged into my Linux VM to update the site? Other than the first time? None. Well, actually once. I tried, but gems had been changed and I had dependency issues and… I just let it sit.

I looked at Pelican first, because I am very comfortable with Python. And Python is 80% as nice in Windows as Unix, maybe more. People actually care if it doesn’t work in Windows. So I came full circle and am now generating the site in Pelican and hosting on github pages.

The style isn’t how I want it to end up, but back online. It will grow with time, as I add new content and port over the old content.

This whole process started with a outdated Wordpress install on a shared server, leading to compromised sites and blank sites. One thing I’m sure of, it is better than Wordpress.
